场景1：固定集合常量

表示一组固定的、预定义的常量（如状态、类型、模式等），比普通常量更安全、更易维护。

public enum OrderStatus {
  PENDING, PAID, SHIPPED,
  DELIVERED, CANCELLED
}

优势

  • 编译时类型检查
  • 自动补全支持
  • 避免拼写错误

场景2：替代魔法值

避免直接使用数字或字符串字面量（如 1、"A"），用枚举提高可读性和类型安全。

public enum LogLevel {
  DEBUG, INFO, WARN, ERROR
}

// 使用：LogLevel.ERROR
// 代替直接写 "ERROR"

使用技巧

配合switch语句使用，编译器会检查是否处理了所有枚举值。

场景3：附加行为

枚举可以包含方法、字段，甚至实现接口，为每个常量赋予特定逻辑。

public enum Operation {

    ADD {
       public double apply(double x, double y) { return x + y; }
    },

    SUBTRACT {
       public double apply(double x, double y) { return x - y; }
    };

    public abstract double apply(double x, double y);

}

适用场景

策略模式 状态模式 命令模式

场景4-5：单例与多例

单例模式

枚举天然支持线程安全的单例实现

public enum Singleton {
  INSTANCE;
  public void doSomething() { /* ... */ }
}

多例模式

一组有限的、预定义的不可变对象

public enum Color {

  RED("#FF0000"),
  GREEN("#00FF00");

  private String hexCode;

  Color(String hexCode) {
    this.hexCode = hexCode;
  }

  public String getHexCode() {
    return hexCode;
  }
}

场景4-5：单例与多例

优势对比

  • 线程安全：JVM保证枚举实例唯一性
  • 序列化安全：无需担心反序列化创建新实例
  • 防反射攻击：无法通过反射创建枚举实例

7 与框架交互

JPA、Spring等框架直接支持枚举，用于数据库字段、请求参数等。

@Enumerated(EnumType.STRING)
private UserRole role; // UserRole是枚举

何时避免枚举？

  • 常量需要动态变化时（枚举在编译时固定）
  • 常量数量极大或可能频繁扩展
