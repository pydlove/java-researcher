Class常量池

核心特征

  • 位置：存储在.class文件中
  • 性质：静态的、编译期生成
  • 生命周期：与.class文件共存

存储内容

// 包含两大类数据：
1. 字面量（Literal）
- 文本字符串（如"Java"）
- 基本类型值（如final常量）
2. 符号引用（Symbolic Reference）
- 类和接口的全限定名
- 字段名称和描述符
- 方法名称和描述符
墨墨说-Java

运行时常量池

核心特征

  • 位置：方法区（JDK8+在元空间）
  • 性质：动态的、内存级结构
  • 生命周期：与类加载器共存

特殊能力

支持运行时动态添加常量：

String s1 = "Java"; // 类加载时加入
String s2 = new String("Java")
.intern(); // 运行时加入
intern()方法：将字符串对象显式添加到运行时常量池

二者关系

转换过程

编译期 → Class常量池 → 类加载 → 运行时常量池

Class常量池

静态不可变

运行时常量池

动态可修改

关键区别

对比项 Class常量池 运行时常量池
存储位置 .class文件 JVM内存
可变性 静态不可变 动态可扩展
内容差异 符号引用 直接引用

实战示例

字符串常量池

// Case 1: 字面量赋值
String s1 = "Java"; // 直接使用常量池
String s2 = "Java"; // 指向同一对象
System.out.println(s1 == s2); // true
// Case 2: new创建对象
String s3 = new String("Java");
String s4 = new String("Java");
System.out.println(s3 == s4); // false

intern()方法

String s5 = new String("Java")
.intern(); // 加入运行时常量池
System.out.println(s1 == s5); // true

核心要点总结

"静动两池相呼应，编译运行各不同"

记忆口诀

  1. Class池静存文件，运行池动在内存
  2. 符号引用转直接，动态链接靠它行
  3. intern方法显添加，字符串池性能升
  4. 两池配合效率高，JVM优化有妙招
